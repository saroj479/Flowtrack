    Run locally

    Two simple ways to use the real product on your machine.

    Path A

    Local AI with Ollama

    Best for privacy. No API key. Everything stays on your laptop.

    1. Install Flowtrack locally
    2. Install Ollama
    3. Run ollama pull llama3
    4. Open the dashboard and keep model on Auto
    Path B

    Cloud AI with API key

    Best if you want faster responses or already pay for a provider.

    1. Pick provider in the dashboard
    2. Paste API key
    3. Leave model on Auto or choose from the dropdown
    4. Chat or run analysis
    Install

    Developer or power user

    Clone the repo and use the current local-first stack directly.

    git clone https://github.com/saroj479/Flowtrack.git cd Flowtrack bash install.sh

    Why the public demo is separate

    Real tracking stays local until you choose to run it on your own machine.

    That is the correct product boundary right now. The demo helps non-technical users understand the idea without exposing screenshots, logs, window titles, or system permissions to a hosted website.

    Public site: landing page + safe demo only
    Local app: real tracking, local screenshots, real analysis
    Later: optional Google login only when you add a real cloud product

    Flowtrack is local-first. This branch is the public landing site for Vercel.
