# FocusFlow

> **Private, local productivity tracker for Linux desktops.**  
> No cloud. No API keys. No subscription. Your data stays on your machine.

FocusFlow watches which window you are working in and builds an honest picture of your real focus patterns — not just "I used Chrome for 3 hours" but *how many times you switched away from it in that time*.

---

## What it tracks

| Signal | Why it matters |
|--------|---------------|
| Active window title + app | The raw attention data |
| Time spent per window | How long you actually stayed focused |
| App-to-app context switches | The #1 productivity killer |
| Screenshot (grayscale, compressed) | Visual confirmation — purged after 48 h |
| Focus Score (0–100) | Weighted score: time-on-task + switch-rate + deep-work ratio |

## Patterns the AI finds

| Pattern | Description |
|---------|-------------|
| **Rapid Switching** | >5 window changes in 2 minutes |
| **The Rabbit Hole** | One search → 20+ unrelated browser tabs |
| **The Fatigue Pattern** | Switch rate in the afternoon is ≥50 % higher than morning |
| **The Visual Bait** | Social / video sites that trigger the longest distraction chains |

## Project structure

```
focusflow/
├── tracker.py          # Daemon — logs every window change + screenshots
├── analyze.py          # AI analyzer — reads logs, scores focus, queries Ollama
├── install.sh          # One-shot installer (venv + systemd user service)
├── focusaudit.service  # systemd unit template (reference copy)
├── requirements.txt    # Python dependencies (mss, Pillow)
└── .gitignore
```

Data written to `~/.focusaudit/` (never committed):

```
~/.focusaudit/
├── logs/           YYYY-MM-DD.jsonl   ← kept forever (tiny text)
├── screenshots/    YYYY-MM-DD_*.jpg   ← auto-deleted after 48 h
├── reports/        analysis_*.txt     ← generated by analyze.py
└── tracker.log
```

## Requirements

### System (Ubuntu / Debian)
```bash
sudo apt install xdotool scrot python3 python3-venv
```

### Other distros
| Package | Fedora/RHEL | Arch |
|---------|-------------|------|
| xdotool | `dnf install xdotool` | `pacman -S xdotool` |
| scrot | `dnf install scrot` | `pacman -S scrot` |
| python3-venv | included | included |

### Optional — local AI (Ollama)
```bash
curl -fsSL https://ollama.com/install.sh | sh
ollama pull llama3        # ~4 GB download, runs fully offline
```
If Ollama is not installed, `analyze.py` prints a ready-to-paste prompt for any web AI (ChatGPT, Claude, Gemini).

> **No API keys required.** Everything runs locally.

## Installation

```bash
git clone https://github.com/YOUR_USERNAME/focusflow.git
cd focusflow
bash install.sh
```

`install.sh` will:
1. Install missing system packages via `apt`  
2. Create `~/.focusaudit/` layout  
3. Build a Python venv and install `mss` + `Pillow`  
4. Copy scripts to `~/.focusaudit/`  
5. Register and enable a **systemd user service** that starts on every login

## Usage

### Check the tracker is running
```bash
systemctl --user status focusaudit
```

### Watch the live log
```bash
tail -f ~/.focusaudit/logs/$(date +%Y-%m-%d).jsonl | python3 -c "
import sys, json
for line in sys.stdin:
    e = json.loads(line)
    print(f\"{e['ts']}  [{e['event']:8}]  {e['app']:20}  {e['title'][:60]}\")
"
```

### Run the AI analyzer
```bash
# Requires Ollama running locally
focusaudit-analyze

# Analyse the last 3 days only
focusaudit-analyze --days 3

# Skip Ollama — print stats and a copy-paste prompt
focusaudit-analyze --no-ai

# Use a different Ollama model
focusaudit-analyze --model mistral
```

### View screenshots
```bash
ls -lh ~/.focusaudit/screenshots/
eog ~/.focusaudit/screenshots/   # GNOME image viewer
```

### Stop / restart the daemon
```bash
systemctl --user stop focusaudit
systemctl --user restart focusaudit
```

## Understanding the Focus Score

$$\text{Score} = 0.4 \times T_{avg} + 0.4 \times S_{rate} + 0.2 \times D_{ratio}$$

| Component | Meaning | 100 = |
|-----------|---------|-------|
| $T_{avg}$ | Average time per window | ≥5 min per window |
| $S_{rate}$ | Switch-rate penalty | 0 switches/hour |
| $D_{ratio}$ | Deep-work ratio | All time in ≥25 min blocks |

A score of **70+** is healthy focused work. Below **40** indicates fragmented attention.

## Privacy

- All data stays on your machine in `~/.focusaudit/`
- Screenshots are automatically **deleted after 48 hours**
- Text logs are kept indefinitely (they are tiny — ~1 KB/day)
- Nothing is uploaded anywhere
- Window titles are stored as plain text — if a title contains sensitive info (e.g. a banking URL), it will appear in the log. You can exclude apps by editing the `EXCLUDED_APPS` set at the top of `tracker.py` *(see Customisation below)*

---

## Customisation

Open `~/.focusaudit/tracker.py` and adjust the constants at the top:

```python
LOG_INTERVAL  = 30      # seconds between "still here" interval entries
PURGE_HOURS   = 48      # delete screenshots older than this
IMG_WIDTH     = 1000    # screenshot resize width in pixels
IMG_QUALITY   = 60      # JPEG quality (lower = smaller files)
```

To exclude specific apps from logging:
```python
EXCLUDED_APPS = {"keepassxc", "gnome-keyring"}
```
*(add this set to tracker.py and add a guard at the top of the main loop)*
